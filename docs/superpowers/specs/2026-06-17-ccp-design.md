# ccp — Claude Code Profiles 设计文档

## 概述

ccp (Claude Code Profiles) 是一个 Rust TUI 工具，用于管理 Claude Code 的多提供商配置。用户通过 TUI 管理 API 配置（环境变量键值对），工具自动将启用的配置生成为 shell 包装脚本到 `~/.local/bin/`，实现在终端中通过 `ccp-<配置名>` 直接启动对应提供商配置的 Claude Code，且完全兼容 Claude Code 所有 CLI 命令。

## 技术栈

| 层 | 选型 |
|---|---|
| TUI 框架 | `ratatui` + `crossterm` |
| 配置格式 | TOML（`toml` crate） |
| 错误处理 | `color-eyre` / `anyhow` |
| 配置文件 | `~/.config/ccp/profiles.toml` |
| 包装脚本 | `~/.local/bin/ccp-<name>` |

## 配置模型

配置文件 `~/.config/ccp/profiles.toml`：

```toml
[[profiles]]
name = "deepseek"
enabled = true
group = "日常"

[profiles.vars]
ANTHROPIC_AUTH_TOKEN = "sk-xxxx"
ANTHROPIC_BASE_URL = "https://dashscope.aliyuncs.com/apps/anthropic"
ANTHROPIC_MODEL = "deepseek-v4-flash"
```

- name：唯一标识，仅允许小写字母、数字、连字符
- vars：用户自定义环境变量键值对，运行时全部 export
- 所有字段由用户自由填写，不做字段约束

### 操作流程

1. **a** — 新增配置，弹出表单填写 name / group / 环境变量键值对
2. **e** — 编辑当前选中配置
3. **Space** — 切换启用/禁用状态
4. **d** — 删除配置（带确认）
5. **/** — 搜索过滤

### 自动同步

所有变更（增/删/改/启用/禁用）实时同步：
- 写 `~/.config/ccp/profiles.toml`
- 更新 `~/.local/bin/ccp-<name>` 包装脚本
- 禁用的配置：删除对应脚本
- 启用的配置：生成/更新脚本

## CLI 命令

```
ccp                    # 打开 TUI
ccp list               # 命令行列出配置
```

无 `ccp install` 命令 — TUI 自动同步，不需要手动操作。

## Wrapper 脚本生成

每个启用的配置生成 `~/.local/bin/ccp-<name>`：

```bash
#!/bin/bash
# Generated by ccp — edit in TUI, changes here will be overwritten
export ANTHROPIC_AUTH_TOKEN='sk-xxxx'
export ANTHROPIC_BASE_URL='https://dashscope.aliyuncs.com/apps/anthropic'
export ANTHROPIC_MODEL='deepseek-v4-flash'
exec claude "$@"
```

- 使用单引号包裹值，避免特殊字符问题
- `exec claude "$@"` — 完全透传所有参数
- 脚本生成后 `chmod +x`
- 覆盖已有脚本时提示用户

## 错误处理

- `~/.local/bin/` 不在 PATH 时，启动 TUI 时右下角提示并提供修复命令
- 写入权限不足时提示用户
- 配置 name 冲突时保存报错
- 同名配置不允许添加

## 非目标

- 不管理 Claude Code 运行时进程
- 不做 API 协议转换
- 不处理 Claude Code 本身的配置（settings.json）
- 不支持密钥加密存储（V1 版本明文存储，权限 600）
